Project overview

As someone who enjoys practicing MMA, I felt like I needed a way to make my journey more efficient. As a result, I started working on this web application to track my progress and get a bigger picture of my journey as a whole.

Technical details

I had decided on a very modular structure for my project in order to facilitate the scalability, readability and management of the code.

File Structure: 

├── Assets
│   ├── arrow.svg
│   └── ... (and other assets)
├── index.html
├── main.js
├── modules
│   ├── cardSorter.js
│   └── ... (and other modules)
├── notes.txt
├── pages
│   ├── cardCreator.css
│   └── ... (and other page files)
└── style.css

Modular JS:

Each module has its own function that contributes to the architecture of the web application. For example, the cards Peter.js module sorts the cards into different categories to later be displayed by another module. In this way, the debugging process is much faster due to different problems being handled by specific files.

Grouped Assets:

All assets are grouped into one folder. This allows us to focus on the code, making things more readable and structured.

For more details about the inner workings of the project, check the GitHub repo: https://github.com /VirtuosoEB/MMAskillTracker
